How to Add App Preference Category
Technical Article

Abstract

The New Preferences command enables an end user to customize preference categories in order to add nodes specific to a CAA App.  This article explains how to add your own CAA node into the "All Preferences" tree.

New Preferences command Overview

The "New Preferences" command gathers in one single dialog box all the app properties that can be customized and made persistent in the user's settings files.

The command is launched from the Me Services->New Preferences menu in the top bar, as shown in the picture below:

The  New Preferences dialog box is organized in two parts:

-       On the right hand side “Preference Categories” are displayed the apps classified by category: “Main Preferences” and “All Preferences”.

-       On the left hand side “Preference Pages” is the page where all settings associated to the selected category are displayed.

When an app is selected, the preferences pages associated with this app are displayed on the right.

Each preferences page displays controls to set parameter values or working options for a given property.

The tab “Main Preference” is dedicated to global settings not specific to an app and then allows specifying General, Display and Modeling settings.

It gathers preferences pages common to all apps.

 “Preference Categories” Organization

Preference Categories purpose two level of access to preferences:

  • "Main Preferences" tab provides a small number of selected preferences which are the the most used and the most important ones.

  • “All Preferences” tab is dedicated for management of settings specific to each app. It regroups All preferences based on the Compass quadrants and apps groups.

 

Working with new Preferences panel

The New Preferences command is accessible from 3DEXPERIENCE only if you launch your session  as following:

  • set NewToolsPreference=1

  • mkrun

Creating New CAA Preference Categories

For the two CAA apps "CAA Analyze Geometry" and "CAA Create Geometry", below is detailed how to create for each one a specific node category defined under a common CAA node under  node "3D Modeling/3DEXPERIENCE Open".

Create a Directory for XML files

 Create the directory ...\resources\CAfrComponents\CAAPreferences

 

This directory will contain the xml files describing the collapsible sections that will be embedded by preference categories associated to CAA Apps.

Create Resources files for description of new nodes

The location of the CAA category should be defined in a preference file where is specified the path in the tree of the collapsible section. The information is provided in CAfrCAAPreferences.CATRsc file with PreferencePageXMLName tag  :

  • PreferencePageXMLName.Location=“All_Preferences\3D_Modeling\3D_Modeling_Core”;  (Existing Node)
  • PreferencePageXMLName.Location=“All_Preferences\3D_Modeling\3D_Modeling_Core\CAATestNode”; (New Node)

The name of the CAA App node will be specified in the CAfrCAAPreferences.CATNls:

In Short

To create a new Preference category in the All Preferences tab, you will need to create a directory specially dedicated to contains all xml files describing callapsible sections that will be associated to caa  nodes.

To specify new node name location and associated preference pages, two resource files must be created:

  • a CATRsc file for each collapsible section detailing the path of the node.
  • a CATNls to specify the public name of the CAA App node.
