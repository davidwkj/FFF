Abstract

This article explains the steps involved in configuring the product structure. An overview of configuration modeler is detailed in technical article [1]. This article describes different operations which can be performed in configuration modeler.

  • Basic Objective
  • Creating a model dictionary with variant configuration central
  • Open/Create a new product
  • Attach the Configuration dictionaries to the root product
  • Edit the Effectivity of an instance

Basic Objective

The basic objective is to make functionalities available to user. So that user will be able to manage the configuration entities on their own.

Creating a model dictionary with variant configuration central

Please refer to Product Doc for details of creating model on Web Application.

Open /Create a new product

configuration can be applied on the existing or the new product, but it is required that the desired product is loaded in VPM Editor workbench session. If user want to configure the existing product then open the desired product in VPM Editor workbench else user can apply this operation by creating a new product.

Attach the Configuration dictionaries to the root product

Configuration Dictionaries or models should be created at the variant configuration central. Select the product you want to configure and select the properties. Configuration is the special property of the product structure. In the configuration tab of properties window search for the dictionaries, attach the desired context and enable the Configuration criteria.

These steps are detailed in Introduction to Configuration Dictionary [1]. Multiple model dictionaries can be attached to the single product.This is the most important step before applying any configuration operation.

Edit the Effectivity of an instance

After attaching the configuration dictionaries to the product structure, select the desired instance, edit the effectivity for an instance from the option available in contextual menu. Effectivity can be editable in the "Define Effectivity" panel.

These steps are detailed in Introduction to Effectivity[2]. Once effectivity is defined user can set the same effectivity for session also. Effectivity can be seen in "Effectivity" tag of Properties window.
