Introduction To Configuration Dictionary

Technical Article

Abstract

Configuration Context consists of a set of dictionaries defining the criteria that can be used to configure the structure. This article describes about concept of Configuration Dictionary and how it is used in following sections :

  • Introduction
  • How to attach configuration dictionary?

Introduction

Configuration Dictionary is the set of multiple criteria as configuration features, product states, units, manufacturing plans. This dictionaries can be attached to the product reference to create variants of product or for evolution of product. In the context of variant configuration central dictionaries are termed as "Model". Model gives the information about all the builds created for a product or the product configuration managed by that model. Multiple dictionaries can be attached to a single product reference. Models manages the products. Multiple models can be managed by a Product Line.

Models control the unit or a range of builds created for the product. It helps to uniquely identify the build by assigning a unit number to each build across all revisions of the managed product.

In configuration central, model contains the product revision these product revision aggregates configuration features. Different builds, manufacturing plans are aggregated under model. These components can be used to configure the product. How different components of model are stored in database is shown in Fig.1.

Fig.1 Model

All these entities are of ER type. Numeric values written at relationship shows the cardinality of this relationship.

1...* : This relationship is of type one to many. This means at least one object should exist. Model should have at least one product associated with it. To achieve this when model is created one product revision "A" is associated under it.

0...* : This relationship is of type many to many. This means multiple entities can exist, this also includes zero. Product revision can have zero to many configuration features under it.

How to Attach Model Dictionaries

This task describes how to assign a model dictionaries to existing object by using properties.

  1. Open the existing product structure in VPM Editor Workbench. Go to properties. In the properties dialog box select Configuration tab to specify the desired configuration criteria.
  2. Click Attach Model as Configuration Context to retrieve the model from database specify the search criteria in "Search For model" dialog box. Select the model to be assigned and click OK. Models are created by using variant configuration central. Refer to Product Doc for details. Attached model is visible in the Context section of the dialog.
  3. Select criteria for configuration. This criteria is used for the child instances to configure. Configuration criteria is classified as variant or evolution.
    Fig.3 Assigning Configuration Dictionaries to the product

    Table at the bottom of above dialog box represents  the predefined configuration.

    If user want to detach a model, then select a model from the context and click on Detach Model

    Note: The model dictionary will be removed from the selected product , even if Close in Properties dialog box is clicked while attaching a model.

For details about assigning models to product structure refer to product documentation.

Once models are assigned to the product, it can be configured. steps for Configuring the product structure are given in "Description"[1]

In Short

Configuration dictionary is the set of multiple configuration criteria which are used for configuring the structure. Configuration criteria consist of configuration features, product states, builds, manufacturing plans. These criteria can be used for generating variants of a product or for evolution of a product. Product can be configured only if model dictionaries are attached to it. Configuration dictionaries creates major difference between Product and configured product.
