Introduction

Effectivity is an expression composed of configuration variables (such as options, specifications) combined with operators (such as AND, OR and NOT). It is a criteria that specifies the availability of an instance for business processes. This conditions could be based on either date or related objects.

Multiple model dictionaries can be attached to product reference. All attached dictionaries collectively called as "Configuration Context". Each configuration of a reference is defined for only one single model. Product configuration is the unique list of features and options for a product that is ordered or selected by a department or a customer. But effectivity is defined at the most in the scope of all the models of a reference. This is represented in Fig.1 :

Fig.1 Configured Product Structure

Each effectivity points to all the criteria defined in the expression using Semantic Relationship. "Semantic Relationship enables that the object express facts that include their own meaning" these are the special kind of relationship which enables applications to interpret the meaning from it's content. Each Effectivity contains a binary describing it's expression.

What is the Effectivity ?

Effectivity expression is the composed of configuration variables combined with logical operators. It is defined in the scope of all the attached models.

Generic Effectivity expression is formed of 1 Global Section, N number of Model Section(N -> Number of attached models) and 1 change stack section.  Formation of generic effectivity expression is shown in Fig.3 :

Fig.3 Effectivity Expression : General Structure

As shown in Fig.4  Generic Effectivity Expression is divided into 3 sections :

  1. One Global Section : In this section, if changes are made under the workspace then Name of workspace is included. This sections also includes the History of changes  i.e. date on which changes are promoted. This section is completely managed by the system. This section is present only in the case changes are associated to workspace .
  2. N Model Sections : Multiple Model dictionaries can be attached to a product reference. This section combines effectivity in the context of each attached model.
  3. One Change Stack Section : This is the expression of applicability, if the product is modified under a change object (ECA).
Fig.4 Generic Effectivity Expression

How To Define the Effectivity Of an Instance

Effectivity is the availability of the instance for business processes. Effectivity can be defined for the instance which is aggregated under the product reference to which the configuration dictionaries are attached. For details about attaching model dictionaries to the product reference refer to  "Introduction to Configuration Dictionary[1]"

Once desired models are attached to the product, insert the product under it. click on the instance for which effectivity should be defined and select "Edit Effectivity of an Instance" from the contextual menu. following "Define Effectivity" panel will appear. Local area in this panel represents the local effectivity expression in matrix format. This matrix can be modified depending on the criteria selection such as product state, manufacturing plan, unit(builds) or configuration feature.

Fig.6 Define Effectivity Panel

If user want to have the same effectivity for next operations (same as newly created product) ,enable the checkbox for "Keep this Effectivity for next operations in session".

How To Retrieve the Effectivity Of an Instance

Once effectivity is defined for an instance user can retrieve the effectivity by following way :

1) select the instance for which effectivity should be retrieved.

2) select Properties of the instance.

3) From the properties window select "Effectivity" tag.

In Short

Effectivity defines the availability of an instance. In the process which are the characteristics are available is mentioned by Effectivity Expression. This Expression is the combination of configuration variables and logical operators like AND, OR. This expression also consist of date history if the changes are made under the scope of workspace. This can be represented in the XML format. The configuration modes describes about the ways available to make the configured operations in authoring session.
