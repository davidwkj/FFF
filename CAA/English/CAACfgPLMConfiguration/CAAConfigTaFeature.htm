Introduction

Features are vehicle for capturing & managing engineering attributes of the product. To represent certain features with enough details feature specific attributes are required. These attributes defines requirement or behavior of Feature. Features are classified based on the type of Product that company develops as Below :

  • Configuration Features :  It represents the Higher-Level marketing requirement and configuring options.
  • Logical Structures :  It includes the Logical Features & Manufacturing Features. Logical Features include Software Features which are used to generate the BOM, While Manufacturing Feature defines how the product is to be manufactured.

Classification Of  Configuration Features

Configuration Features are further classified into :

  • Configuration Feature : It addresses the Marketing Requirement. e.g. Fuel capacity and type of fuel are the configuration feature for engine.
  • Configuration Option   : It defines the actual choices available under the configuration Feature. These are Leaf - Level Choices available in specific feature to configure the product and cannot be further broken down.  e.g. When fuel type is configuration feature,available configuration options are Petrol, Diesel etc . If fuel capacity is configuration feature, configuration options are 35L, 40L etc.  

Feature Lifecycle

Configuration Features goes through following lifecycle stages :

  • Preliminary : This state defines the initial condition for a feature object. Users with a Product Manager or System Engineer role are most likely to define the configuration feature composition while it is in this state.
  • Review :  Configuration features are promoted to this state upon initial composition definition.The Responsible Design Organization (RDO) governs all role definitions in this state, and role definitions have RDO filters in their policy. User/ Design engineer can use defined feature only if it is in review state.
  • Release : Configuration features are promoted to this state using the change control behavior of the Engineering Change object. You cannot manually promote or demote from this state, which is considered a frozen state for the configuration feature. It is not possible to make any change to the composition of a configuration feature when it is in this state; however, it is still possible to define a GBOM. The configuration feature is not governed by the RDO when it is in this state.
  • Obsolete : Configuration features are promoted to this state using the change control behavior of the engineering change object. You cannot manually promote or demote from this state, which is considered a frozen state for the configuration feature. It is also not possible to make any change to the composition of a configuration feature when it is in this state. The configuration feature is not governed by the RDO when it is in this state.

How To Create Configuration Feature

You can create the configuration features by following steps :

    We can create a stand-alone configuration feature that can be reused across multiple contexts using the add existing option.
  • To create a configuration feature in the context of a product, product line, product variant, product version, or configuration feature, from the Configuration feature page select the desired context object, click Enable Edit, then from the Actions menu, select "Create Configuration Feature".
  • In the context of a model, from the configuration features page, on the candidate configuration features tab, from the page actions menu, select "Create Configuration Feature".

Detailed description of creating feature refer to product documentation.

Navigation in Product Structure

Configuration features are associated with the product revision. Different product can share same configuration features and options. Configuration feature are used as variant criteria.

In case of manufacturing plans this is not the case, same manufacturing plan cannot be used for different products. Therefore it is aggregated under the model. Manufacturing plans are used as evolution criteria.

Configuration Feature : Variant Criteria

Configuration is the process of  managing the variants. This is achieved by using configuration feature. Each product is having multiple features( characteristics) by using this features multiple variants can be created.

Configuration Options are aggregated under the configuration features. Configuration options stores actual information about characteristics of product.

e.g. while manufacturing car, engine is a commonly used product. There are different types of engine depending on the type of fuel. so for car "Fuel Type" will be treated as the configuration feature. Actual values for the type are Petrol,Diesel(values are depending on the model and country) so these values are treated as configuration options for engine fuel.

In this case, different variants for car are : CAR Eng. Fuel-Petrol, CAR Eng. Fuel-Diesel

In Short

Configuration Feature achieves the management of product variants. Manufacturing plans/ features are used as evolution criteria. This is the essential component as it makes possible to generate the different variants of product. It can exist as stand-alone object or the object in the context of some product or model.
