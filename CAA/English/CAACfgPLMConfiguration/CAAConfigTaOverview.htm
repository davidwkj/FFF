Configuration -An Overview

Technical Article

Abstract

Configuration is the process of applying the set of parameter values to the object. System Engineers and Product Managers can use configuration to define a portfolio of products and features with configuration options based on specific customer and market requirements.

This article provides an overview of Configuration in following steps:

  • What is Configuration?
  • Why Configuration is required ?
  • Who applies and uses Configuration ?
  • When and Where  Configuration can be applied ?
  • Overall Architecture
  • Configuration Openness
  • To Conclude
  • References

What is Configuration ?

In general Configuration means the way how the things are arranged or put together in order to achieve a result. In PLM domain, Configuration enables the companies to manage the conceptual and commercial aspects of their products. It contains the functional and physical characteristics which are defined in a technical document and achieved in product.

Why Configuration is required ?

It is very important to ensure all the parties to the acquisition of product :

  • User
  • Buyer
  • Developer / Designer
  • Producer

Has the common understanding about :

  • How the product looks like ( Physical Shape )
  • What it supposed to do ( Functionality , Performance )
  • How it can be operated ( Capabilities )

To generate the desired product, it is necessary that manufacturing process follows the approved and correct documentation. Configurations enables validation of manufacturing process at any moment in the lifecycle of product. Comparison of the desired configuration and built in configuration helps to identify the discrepancies detected during production, delivery or operation of the product. It also helps user to know about the operational capabilities and limitations of the product.

Who applies and uses Configuration ?

Product Managers and System Engineers can create the sample configuration. A design variant or the Configuration Features must be associated with this configuration. This configuration features addresses the marketing requirements. once base functional product is defined, product variants can be generated.

When configurations are successfully specified and validated the system creates a custom product for company. Anyone else who is able to log into the application in company can order and review these custom configurations.

Manufacturing and Design Engineering can use this Engineering BOM (EBOM) as a starting point to design the new configuration.

When and Where  Configuration can be applied ?

Configuration is the "Special Case of Design Activity", System Engineers generate the sample configurations. 

At any point in the lifecycle of the product to validate the structure and rules of compatibility this sample configurations are used. Engineers can use this sample configurations to test and validate the Structure and rules. Depending on this report Precise Bill of Material(PBOM) can be generated. PBOM is the list of all parts required to manufacture the defined configurations.

validated sample configuration is a custom for company. Further Design Engineers can use these validated configuration to generate new configuration.

Overall Architecture

Configuration work on two different data models, which are as follows:

  1. Variant Configuration Central :

    System Engineers or Product managers can define the model to describe the line of product that particular company offers. Configuration Context or Model Dictionary is the collection of Configuration Features, Configuration Options , Manufacturing plans, Product State and Units.

    This model can be created  through Variant Configuration Central. For detailed description refer to product doc.Configuration Dictionaryis explained in "Introduction to Configuration Dictionary"[1].

    Once Configuration context is ready then certain member of organization with design responsibility can begin creating product configurations. This sample configurations are validated by using the structures and rules which are defined in the earlier steps to generate the PBOM. Once all the parts are verified and corrected  the EBOM can be generated. Variant configuration central automates the process of generating Engineering BOM.

  2.  VPM Editors :

    3. Design Engineers can use this Engineering BOM as a starting point to generate new configurations. This can be done on VPM (CATIA) side. For detailed description refer to Product Doc , " Description "[2].

Configuration Openness

Now configuration capabilities are available only through VPM editors. Here we are providing browsing and navigating configuration objects. In addition to navigation, we also provide some functionalities for accessing the effectivity associated with PLM instances.

Browsing of configured product includes opening and loading attached dictionaries in session. Navigation of configured product structure refers to retrieval of the criteria attributes.

Configuration criteria includes following components :

  • Configuration features
  • Configuration options
  • Manufacturing plans
  • Product state
  • Builds/Units

Main objective of this task is to make functionalities of configuration avaialbe through CAA APIs so that user can do further customization with it.

To Conclude

Configuration process the industry can assure about the product consistency. It assures to deliver the identical items, allow to modify the product without the need of reverse engineering. It gives the relief from high cost considerations caused due to change in product by identification of affected changes at design stage only. Proper identification of approved changes and controlled cost allows least cost of maintenance and reuse. This also helps to improve the process of manufacturing to manage the phases which are affecting the product lifecycle.
