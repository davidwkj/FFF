Introduction to Predefined Configuration

Technical Article

Abstract

Predefined Configuration is the criteria that will identify the suitable elements for a configuration based on their effectivity. According to this ,it is a list of selected variant and evolution criteria for a given model. 

This articles describes about the predefined configuration in following steps :

  • Introduction
  • How to Create the Predefined Configuration ?
  • Illustrating a Predefine Configuration with Example
  • In Short
  • References

Introduction

Predefined Configuration is a filter that defines a specific view of a configured structure for a given configuration definition. This identifies the suitable product elements for a specific configuration based on their effectivity in a configuration controlled process. This is a persistent filter which allows easy navigation and opening the required configuration.

How to Create Predefined configuration

For a product new predefined configurations can be created. This functionality is available in the authoring window.

New predefined configuration can be created as follows:

  1.  Attach Model Dictionaries to Product Structure :

    To apply any configuration operation it is mandatory to attach the model dictionaries to the product reference. This can be done through the configuration tab in properties window. Details are given in "Introduction to configuration Dictionary"[1] .Table on the bottom of properties dialog represents the Predefined configuration.

  2. Create New Predefined Configuration :

    Click "New Predefined Configuration" .New-Predefined Configuration-Properties dialog box will appear. Click "Next" to set the selected criteria.

  3. Set the Criteria :

    New-Predefined Configuration-Expression dialog Box will appear, as shown in Fig.1. select the attached model from the context drop down located at the top of dialog. Criteria section displays the available variant or evolution criteria for selected model. Select the criteria and click on Next.

    Fig.1 Create New Predefined Configuration
  4. Displaying the New Predefined Configuration:

    New Predefined Configuration-Summary dialog box will appear. This dialog box summarize the selected criteria. If user wants to save the criteria in the text file then select the option "Export Configuration Summary To File". This Generates a text file. Newly created predefined configuration is shown in the table at the bottom of the properties dialog box, as shown in Fig.2

    Fig.2 Predefined Configuration
  5. Save the New Predefined Configuration :

    Newly created predefined configuration will not be stored in the database automatically. To save predefined configuration, select the PLM access and click "Propagate"

For more details about the creation of new predefined configuration refer to product documentation.

In Short

Predefined Configuration is the specific filtered view of configured Product. Basic purpose of using configuration filter is easy navigation. Once predefined configuration is applied then anyone can use this structure for other operations. This helps to save the time in applying the effectivity each time. Once the Predefined configuration is created then user(having right credentials) can edit it any time whenever it is required.This article describes about the predefined configuration and how it helps for easy navigation.
