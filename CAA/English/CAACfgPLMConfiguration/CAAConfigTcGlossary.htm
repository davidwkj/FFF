Abstract

This article describes glossary for Configuration.

Glossary for Configuration

Configuration In Knowledgeware, a consistent set of parameter values that can be applied to a document. A customizable grouping of data created by displaying properties from one or more related database classes. Context consists of a set of dictionaries defining the criteria that can be used to configure the structure. e.g. Multiple views, also known as forms, for a screen may be constructed based on user organization preferences and requirements.
Product Configuration In ENOVIA Variant Configuration, a unique list of features and options for a product that is ordered or selected by an internal engineering department or by a customer.
Configured Product Structure A product structure with the instances having effectivities assigned to these instances with the root product (context) associated to a variability space.
Model Dictionary / Model Objects These are dictionaries that can be assigned; it contains multiple criteria as Configuration Features, Product states, Units and Manufacturing Plans .
Configuration Features A Set of Configuration Features ( Defines the list of options that can be activated (or not) for a given product.)
Configuration Feature A type of ENOVIA feature that addresses marketing requirements. The Configuration Features area contains the configuration options associated with the product states to the product revision of the model.
When this configuration criteria type is applied on the parent reference, the Configuration Features area is available. You can select the configuration features from the Configuration Features area and the Local area is populated with your selections.
Configuration Features allowing multiple selected values is symbolized as and Configuration Features allowing only one selected value is symbolized as . This information is displayed in both Criteria and Expression section.
Configuration Option Multiple options availble in specific Feature to Configure the product
Manufacturing Plans A set of process steps that describe how an assembly is manufactured. The Manufacturing Plans area contains the different manufacturing plan created for the model. When this configuration criteria type is applied on the parent reference, the Manufacturing Plans area is available. You can select the manufacturing plans from the Manufacturing Plans area and the Local area is populated with your selections. Defines the different stages of the manufacturing process for a given product.
Units The Units area contains the different builds created for the model. Each build contain units number that can be used to define the effectivity.
When this configuration criteria type is applied on the parent reference, the Units area is available. You can select the builds from the Units area and the Local area is populated with your selections.
Defines the different unit numbers that are built for a given product.
Product States The Product States area contains the different product revisions created for the model. It defines the different stages of the configuration definition for a given product. This is Part of a lifecycle that determines who can access and perform specific tasks on an object. A CAD object usually goes through these states in its lifecycle: Preliminary, Review, Approved, Release.
When this configuration criteria type is applied on the parent reference, the Product States area is used for selecting the criteria. The Local area available under the Expression section is populated with your selections.

Note: If this configuration criteria type is not applied on the parent reference, the Product States area is still available and used as a filter for the availability of other criteria. In this scenario, the Product States cannot be added in the Local effectivity, but are only used to filter the available Configuration features.
Product Revision In ENOVIA Product Line Management, a change to the original product, where all related features are retained by default. A product can be revised at any state in its lifecycle and multiple unreleased revisions can exist at the same time.
Variant It is variation of an existing product that shares some the existing product's features, but not others. It is a term used to define the characteristics of a product defined using option or specification.
configuration rule A rule that determines which features or products can be included or what must be excluded from a product configuration.
Criteria List of configuration criteria types available in the model dictionaries that are attached to the product. (Criteria are classified as either Variant or Evolution. )
Effectivity A criteria that specifies the availability of an instance for business processes. The conditions could be based on either date or related objects. It is an expression composed of configuration variables (such as options, specification or ranges) combined with operators (such as AND, OR and NOT). There are two types of Effectivity : 1) Inherited 2) Local
Inherited Effectivity The Inherited Effectivity area lists the instance name of the inherited effectivities along with its effectivity expression.
Local Area The Local area lists the local effectivity expression in matrix format. This matrix can be modified depending on the criteria selection such as Product State, Manufacturing Plan, Unit or Configuration Feature. One or more configuration options are available for each configuration feature.
Expression An object (persistent or not) used to describe with a text an evaluation method and that can be used to perform the evaluation on parameters. It enables to directly view and edit the matrix layout once the values are added
Effectivity in XML Expression Effectivity setted in the XML format for visibility purpose .
