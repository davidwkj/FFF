How to Launch Convertor Services Use-case
Use Case

Abstract

This article describes the steps to perform in order to execute the convertor services code after launching 3DEXPERIENCE Platform

Steps to Perform

  1. Launch 3DEXPERIENCE Platform.
  2. Select '+' sign | Import | Select 'Format' from the drop-down list <own extension> (as shown in image below)

    3. To get your own extension file option in CATIA File-based Import , you will need to set up the build time environment, then compile CAAVIConvertors.edu framework along with its prerequisites, set up the run time environment, and then launch 3DEXPERIENCE Platform.

    The use-cases are made up of a DLL files, so they will get loaded after you select a file of your desired extension and click on 'OPEN'.

    Fig. 1 steps to launch the convertors use-case

    Here, the extension .CAAcmp and the usage MyXCAD_CAAVIPoint are customizable through changes in .capa file. Please refer to the article[1] for more information

  3. Browse the 'Filename' where files reside for the format you selected.
  4. Available file with the specified format will be listed as shown in image below.
Fig. 2 Import from file

At last when you click on OK, the convertors code is called and our DLL's gets loaded. The flow of the execution code is explained in use-case articles.

Note: The file to be selected in the Import step are specifically mentioned in the use-case articles. Please select those files to run a particular use-case scenario
