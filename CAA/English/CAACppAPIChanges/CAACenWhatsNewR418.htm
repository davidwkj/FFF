What's New in 3DEXPERIENCE R2016x
Technical Article

Abstract

New modelers, or enhancements to existing modelers are listed here, compared with the previous release. Some enhancements or maintenance reasons may also require changes regarding tools or methodologies. This article lists them, and points to the corresponding documentation.

Toolbar Deprecation

Since the 3DEXPERIENCE release, the layout of commands in the frame has completely changed. The action bar has become the main location for app commands, replacing toolbars. This change is not transparent for your own apps or add-ins, and a quite easy migration is required to insert your old toolbars in the action bar. Until complete deprecation, the toolbars are still visualized. Their appear by default below the action bar. The deprecation of toolbars will lead to not displaying the toolbars at run-time anymore, but will not lead to any build errors.

The following articles explain the user interface novelties and how to migrate your apps and add-ins:

  • IFWE Compass User Experience Novelties
  • IFWE Compass User Experience Adoption Tips
  • Example of Migration for the CAA Geometry Sample

Declarative Application Frame

Declarative Application Frame or D-Afr is a new infrastructure that enables you to create apps and add-ins by using a declarative syntax based on XML rather than using only C++ code, thus making the development of apps and add-ins easier and more flexible.

The C++ classes that can be replaced by a declarative file are the classes that declare the command headers and their layout in the frame.

  • In the case of an app, the class that implements the xxxConfiguration interface provided by the data model on which the new app is built.
  • In the case of an add-in, the class that implements the xxxAddin interface provided by the app to be extended.

The following articles explain how to create apps and add-ins using the D-Afr infrastructure:

  • Creating an App with a Declarative File
  • Creating an Add-in with a Declarative File
  • Creating a Global Add-in with a Declarative File
  • Creating a Data Model Add-in with a Declarative File
