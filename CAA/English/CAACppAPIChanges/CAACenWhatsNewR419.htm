What's New in 3DEXPERIENCE R2017x
Technical Article

Abstract

New modelers, or enhancements to existing modelers are listed here, compared with the previous release. Some enhancements or maintenance reasons may also require changes regarding tools or methodologies. This article lists them, and points to the corresponding documentation.

Introduction of Representation Occurrences

To support new design capability like the ability to hide/show specific occurrences of a representation, occurrences of the representation objects have been added to the Native Client session model.

The API has been kept compatible at best for existing apps by temporarily implementing interfaces supported by representation instances on the newly introduced representation occurrences, allowing a progressive adoption of the new session model. Is is however highly recommended to start this adoption in 3DEXPERIENCE R2017x.

Details on this operation can be found in Adopting Representation Occurrences

New Way to Manage ExportedBy in Header Files

A new syntax is now proposed to manage symbols export and prepare performance optimizations in incoming releases.

Until V5-6R2016, 3DEXPERIENCE R2016x Since V5-6R2017, 3DEXPERIENCE R2017x 
#ifdef _WINDOWS_SOURCE


#ifndef ExportedByToolkitEdit_H
#define ExportedByToolkitEdit_H

    #ifdef __ToolkitEdit
	#define ExportedByToolkitEdit __declspec (dllexport)
    #else
        #define ExportedByToolkitEdit __declspec (dllimport)
    #endif


    #if defined(__ToolkitEdit)
        # define ExportedByToolkitEdit   DSYExport
    #else // __ToolkitEdit
        # define ExportedByToolkitEdit   DSYImport
    #endif  // __ToolkitEdit

#else

    #define ExportedByToolkitEdit

#endif




    #include "DSYExport.h"

#endif //ExportedByToolkitEdit_H

No migration is required, part of the API has already been migrated but it has no impact on existing apps. Guidelines have been updated and code generation wizards now generate the new syntax.
