Abstract

This article provides basic information about installation of X-CAD. It explains about necessary components to be installed for X-CAD integration. Necessary points to be considered for executing use cases are listed in this article.

Installation and Configuration for X-CAD Integration

Installation

To achieve integration of X-CAD and 3DEXPERIENCE Platform we have to install some additional packages on 3DEXPERIENCE Platform server stack. To know more about each component in detail refer to Product Documentation.

Please note that for enabling X-CAD integration user must install ENOVIAIntegrationExchangeFrameworkClient on client machine.

User have to install CAA schema on server stack in order to execute these usecases. Refer to CAA Schema User Guide [1].

Configuration

Once installation is ready user have to perform following operations to configure installation :

  • Create new OCDX user : This user is required as "Administrator" user do not have creator role so it will not able to checkin files.
  • Assign "Integration User" and  "Author" role to newly created user. (User must have "Author" role for using Checkin functionality).
  • Assign OCDX licenses to newly created user. Some additional licenses to be assigned for X-CAD integration are : IFW, CSV, CDR.
  • Assign "Collaborative Space" to newly created user.

Refer to Product documentation for details about new user creation.

Before Launching the Use Case

Before launching this use case, We have to perform following checks :

  • Database is started.
  • 3DEXPERIENCE Platform (V6 server) is running.
  • ENOVIAIntegrationExchangeFrameworkClient is installed on client machine.
  • CAA Schema is installed on server.

In order to executes CAA use cases, user have to launch the MyX-CAD application. Refer to How to launch MyX-CAD [2] for details.

To execute different scenario related to X-CAD, Please refer to X-CAD Add-ins [3].
