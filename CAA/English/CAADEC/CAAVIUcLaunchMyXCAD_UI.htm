How to Launch a MyXCAD Use Case
Use Case

Abstract

An overview of this article to describes the basic steps for launching the MyX-CAD App. It is important to understand this is only sample application, user could create its own application and replace this.

Basic Objective

The basic objective of this use case is that, how to launch the MyX-CAD Application and its User Interface.

Launch MyXCAD

Run the Launcher program from MyX-CAD.

Browse and locate addin jar files, usually these files we could locate at runtime view.

e.g. WS\win_b64\docs\javaserver

Fig.1: MyX-CAD Launcher

In Fig.1 we could see that message for Data Model Found and add-in found:

  • There is data models available (User could add their own data model also)
    • Sample Data

      addin path for this data type CAAVIMyXCAD.edu\SampleData.mj

      This data model/type are explained in subsequent section.

  • Addin Found shows addin available (more addin could be added)
    • ENOVIA V6 addin

      path for this addin CAAVIXCADIntegration.edu\CAAXCADAddin.mj

      This addin are explained in below section

Data models

Data models are the basic building element of the application, in MyX-CAD app we have two type of data model.

Sample Data

Sample data type is only created for this application. This is important to understand that this are only sample data type user can also create its own data type and add its addin to application. Here in this data type we have two sample data type which are:

  • Component - .CAAcmp format
  • Assembly structure - .CAAasm format

The further details for this sample data type are explained in the article[1].

Addins

Addins are the important feature of the MyX-CAD, this are responsible for the attachment of the other app like ENOVIA V6 to the MyX-CAD. Here We have Single addin, this addin are loaded at the runtime of the application.

ENOVIA V6 Addins

When this addin are loaded in the MyX-CAD we get the ENOVIA V6 connectivity enabled in the MyX-CAD App. The functionality of addin to connect the MyX-CAD with the ENOVIA V6. and we are able to easily store, retrieve the file from the Remote server from MyX-CAD app UI.

Next step we have to launch the App after selecting the path for the addin.

We will find the Basic menu for the MyX-CAD App [Fig.2].

Fig.2 MyX-CAD Application

How to use data type in this application are explained in the article[1], and integration of MyX-CAD with ENOVIA V6 is detailed in the article [2].

In Short

We have learn to launch the MyX-CAD App. with proper data addins and data model. We can also add our own data models.
