Checking the existence of Operation Positions
Use Case

Abstract

This article discusses the CAADELMPPosition use case. It explains how to check the existence of Part or Resource Position on an Operation using DELIAMPPosition in Equipment Allocation App.

What You Will Learn With This Use Case?

This use case is intended to help you check if Part Position or Resource Position has been defined on an operation before you go about doing other things with them in Equipment Allocation App.

The CAADELMPPosition Use Case

CAADELMPPosition is a use case of the CAADELPPRAuthFoundItf.edu framework that illustrates Operation Position checking capabilities.

What Does CAADELMPPosition Do?

In this use case user can:

  • Verify if Position for Manufactured Items has been defined on an operation.
  • Verify if Position for Resources has been defined on an operation.

How to Launch CAADELMPPosition?

To use CAADELMPPosition, you will need to:

  • Set up the build time environment, then compile the CAADELMPPosition.m module along with its prerequisites [1].

Where to Find the CAADELMPPosition Code?

The CAADELMPPosition use case code is located in the CAADELMPPosition.m module of the CAADELPPRAuthFoundItf.edu framework:

InstallRootFolder\CAADoc\CAADELPPRAuthFoundItf.edu\CAADELMPPosition.m

where InstallRootFolder [1] is the folder where the API CD-ROM is installed.

Step-by-Step

This implementation for this use case is located in the file CAADELMPPosition.m\src\CAADELMPPosition.cpp. Its purpose is to allow user to check if position for Manufactured Items or Resources is defined on an operation.

There are two logical steps in CAADELMPPosition:

  1. Checking position for Manufactured Items
  2. Checking position for Resources

We now comment each of those sections by looking at the code.

Checking position for Manufactured Items

On the selected Operaion call:

    DELIAMPPosition_var hOperationMPPos(hChildOp);
    if (NULL_var != hOperationMPPos)
    {
        // Part Position Check
        if (TRUE == hOperationMPPos->IsPartPositionDefined())
    }

Checking position for Resources

On the selected Operaion call:

    DELIAMPPosition_var hOperationMPPos(hChildOp);
    if (NULL_var != hOperationMPPos)
    {
        // Resource Position Check
        if (TRUE == hOperationMPPos->IsResourcePositionDefined())
    }
