What You Will Learn With This Use Case?

This use case is intended to help you customize Resource Position functionality in Equipment Allocation App. It consists of:

  • Describing the different steps dedicated to the knowhow of the company through the customization.
  • Using the methods of DELIAMPResourcePosition to add/modify/remove Resource Position in the Equipment Allocation App.

The CAADELMPResourcePosition Use Case

CAADELMPResourcePosition is a use case of the CAADELPPRAuthFoundItf.edu framework that illustrates Resource Position customization capabilities.

What Does CAADELMPResourcePosition Do?

In this use case user can:

  • Define a Position for a resource assigned to an operation.
  • Retrieve the position defined for a resource.
  • Remove position defined for a resource.
  • Remove positions defined for all resources on an operation.

How to Launch CAADELMPResourcePosition?

To use CAADELMPResourcePosition, you will need to:

  • Set up the build time environment, then compile the CAADELMPResourcePosition.m module along with its prerequisites [1].
  • Set up the run time environment.
  • Launch a 3DEXPERIENCE session.
  • Go to the Equipment Allocation App.

Where to Find the CAADELMPResourcePosition Code?

The CAADELMPResourcePosition use case code is located in the CAADELMPResourcePosition.m module of the CAADELPPRAuthFoundItf.edu framework:

InstallRootFolder\CAADoc\CAADELPPRAuthFoundItf.edu\CAADELMPResourcePosition.m

where InstallRootFolder [1] is the folder where the API CD-ROM is installed.

Step-by-Step

The class CAADELMPResourcePositionCmd provides a command. This implementation is located in the file CAADELMPResourcePosition.m\src\CAADELMPResourcePositionCmd.cpp. Its purpose is to allow user to define/manage Resource position on an operation/system. It provides a dialog with possible options that can be performed on the operation.

There are three logical steps in CAADELMPResourcePosition:

  1. Defining position for a resource on Operation
  2. Retrieving the resources positioned on an Operation
  3. Removing position for a Resource on Operation

We now comment each of those sections by looking at the code.

Defining position for a resource on Operation

Select an Operaion and launch the command CAADELMPResourcePositionCmd. Then select a resource that is assigned to the operation and a Manufactured Item implemented by Operation. A dialog is displayed with three sections.

  • To choose API of DELIAMPResourcePosition to call
  • To choose type of Referential
  • To choose Applicability

    • Select AddPosition, Implemented Mfg Item and Operation options in the dialog and click OK. The selected Resource is positioned with position provided to the CAADELMPResourcePosition::AddPosition API.

    void CAADELMPResourcePositionCmd::OnAddPosition()
{
    CATMathTransformation AbsPositionOfRes;
    CATMathVector vecPos(-3381.325, -822.522, -42.391);
    AbsPositionOfRes.SetVector(vecPos);
    HRESULT RC = E_FAIL;
    if (FALSE == bRelativePosition)
        RC = hResPosition->AddPosition(hResNavOcc, AbsPositionOfRes, hReferentialNavOcc, eReferentialType, eApplicability);
    else
    {
        CATMathTransformation RelativePos;
        RC = hResPosition->AddRelativePosition(hResNavOcc, RelativePos, hReferentialNavOcc, eReferentialType, eApplicability);
    }
}

    If it is absolute position of the resource that user wants to set then use DELIAMPResourcePosition::AddPoistion If user wants to position resource relative to a referential then ues DELIAMPResourcePosition::AddRelativePosition These APIs can be called on multiple resources assigned to operation in a loop. It is also the same API to be called if user wishes to modify the existing position of a resource.

    Retrieving the resources positioned on an Operation

    After positioning one or more resources on operation, user can get those resources and their positions.

    void CAADELMPResourcePositionCmd::OnGetPositions()
{
    CATListPtrCATIPLMNavOccurrence ListOfResourceOccs;
    CATMathTransformationList ListOfPtrAbsolutePositions;
    CATListOfInt ListOfApplicabilities;
    CATListOfInt ListOfReferentialTypes;
    CATListPtrCATIPLMNavOccurrence ListOfReferentialOccs;
    HRESULT RC = E_FAIL;
    if (FALSE == bRelativePosition)
        RC = hResPosition->GetAllPositionedResources(ListOfResourceOccs, ListOfPtrAbsolutePositions,
            ListOfApplicabilities, ListOfReferentialTypes, ListOfReferentialOccs);
    else
    {
        CATMathTransformation RelativePos;
        RC = hResPosition->GetAllRelativePositions(ListOfResourceOccs, ListOfPtrAbsolutePositions,
            ListOfApplicabilities, ListOfReferentialTypes, ListOfReferentialOccs);
    }
}

    If it is absolute positions of the resource that user wants to get then use DELIAMPResourcePosition::GetAllPositionedResources. If user wants get relative positions then use DELIAMPResourcePosition::GetAllRelativePositions.

    Removing position for a Resource on Operation

    It is possible to remove a resource and its position.

    void CAADELMPResourcePositionCmd::OnRemovePosition()
{
    DELIAMPResourcePosition_var hResPosition(_hOperation);
    CATIPLMNavOccurrence_var hResNavOcc(_hSelectedResource);
    if(NULL_var != hResPosition && NULL_var != hResNavOcc)
        hResPosition->RemovePosition(hResNavOcc);
}

    Also, it is possible to remove ALL resources and their positions.

    void CAADELMPResourcePositionCmd::OnRemoveAll()
{
    DELIAMPResourcePosition_var hResPosition(_hOperation);
    if(NULL_var != hResPosition)
    hResPosition->RemoveAllPositions(hResNavOcc);
}

    In Short

    This article provides an example on how to use the DELIAMPResourcePosition interface to add/modify/remove positions for resources assigned to operation in Equipment Allocation App.
