What You Will Learn With This Use Case

This use case is intended to show you how to use CATIDECProductToProduct services, to filter a V6 PLM object to an other V6 PLM object.

The CAAProductToProductSample Use Case

CAAProductToProductSample  is a use case of the CAADataFiltering.edu framework that mainly illustrates CATDataExchControlUseItf frameworks capabilities.

What Does CAAProductToProductSample Do

The CAAProductToProductSample use case creates a filtered PLM object from an original PLM Object.

Fig. 1: illustration of how Product Filtering works
  • First, CAAProductToProductSample initialize the PLM environment (V6 session) with the given user parameters.
  • The goal of CAAProductToProductSample is to open a V6 PLM object and filtered to another V6 PLM object.
  • After the creation of the filtered PLM object, the PLM environment is closed.

In case of failure, CAAProductToProductSample prints the occurred error message.

How to Launch CAAProductToProductSample

To launch CAAProductToProductSample , you will need to set up the build time environment, then compile CAAProductToProductSample along with its prerequisites, set up the run time environment, and then execute the use case [1].

To launch the use case execute the command:

mkrun -c "CAAProductToProductSample Repository Server LoginTicket PLMExternalIDProduct Version"

where:

Repository
Server The server as Host:port_number/RootURI
LoginTicket The String representing a Login Ticket
PLMExternalIDProduct Product Reference from the PLM ID
Version ---

Where to Find the CAAProductToProductSample Code

The CAAProductToProductSample use case is made of a single file located in the CAAProductToProductSample.m module of the CAADataFiltering.edu framework:

InstallRootFolder\CAADoc\CAADataFiltering.edu\CAAProductToProductSample.m

where InstallRootFolder [1] is the folder where the API CD-ROM is installed.

Step-by-Step

There are several logical steps in CAAProductToProductSample use case.

  1. Setting up the connection parameters, create the PLM session
  2. Retrieving PLMCoreReference corresponding to the input Product from database
  3. Creating the Filtered Document Object
  4. Closing the session

Setting up the connection parameters create the PLM session

Before the PLM session could be created, we need to specify the connection parameters with CATPLMSessionServices::SetPLMSessionParameter.

First the Repository:

...
	hr = CATPLMSessionServices::SetPLMSessionParameter("Repository", repository);
...

Then the server:

...
	hr = CATPLMSessionServices::SetPLMSessionParameter("Server", server);
...

 the loginTicket parameters :

...
	hr = CATPLMSessionServices::SetPLMSessionParameter("LoginTicket", LoginTicket);
...

Now that all connection parameters are set, we can proceed to create the PLM session and connect to the repository.

Creating the PLM session is simply done by calling CATPLMSessionServices::InitPLMSession.

...
	hr = CATPLMSessionServices::InitPLMSession();
...

As there can only be one session for the runnnig program, the need for a pointer on the session is irrelevant.

Retrieving PLMCoreReference corresponding to the input Product from database

...   
//---------------------------------------------------------------------------
// Open the PLMCoreReference corresponding to the input Product from database
//---------------------------------------------------------------------------

CATIPLMProducts *piInputPLMProducts = NULL; 
CATBoolean ExpandAllMode = FALSE;

hr = ::CAAOpenPLMComponent("PLMCoreReference", comp_externalId, comp_version,
                            IID_CATIPLMProducts, (void **)&piInputPLMProducts, lifeCycleBag, ExpandAllMode);

 ...

Creating the Filtered Document Object

...
//---------------------------------------------------------------------------
// Filter the Product into another Product
//--------------------------------------------------------------------------- 

CATIPLMProducts * piOuputPLMProducts = NULL;

// creating the convertor
CATIDECProductToProduct * pProductToProductConvertor = NULL;
if(piInputPLMProducts)
{
HRESULT rc = piInputPLMProducts->QueryInterface(IID_CATIDECProductToProduct, (void**)&pProductToProductConvertor);
}

if(pProductToProductConvertor)
{
// set options
pProductToProductConvertor->SetOption("Suffix=", TRUE);
pProductToProductConvertor->SetOption("KeepSubColors", FALSE);
pProductToProductConvertor->SetOption("NoKeepSketchSpec", TRUE);

// do the job 
hr = pProductToProductConvertor->Run();
...

// Get the Result
piOuputPLMProducts = pProductToProductConvertor->GetResult();

...

Closing the session

...   

hr = CATPLMSessionServices::ClosePLMSession();

Close the session, quit the PLM environment.
