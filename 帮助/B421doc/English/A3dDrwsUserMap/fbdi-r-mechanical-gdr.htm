Considerations

You should take the following considerations into account:

FBDI Process

When the FBDI process is successful, a few links/items are partially imported. The table above indicates the modified items and their import status. The FBDI process is successful when the CATDrawing:

  • has links to accessible documents only.
  • has links to a non-accessible document but has been isolated (using the Always activated or User choice option via Me > Preferences> File-based Design Import > FBDI General tab, Link Isolation area).

Note: A CATDrawing isolated from 3D has specific behaviors as follows:

  • Inserted objects (OLE) created with link are deleted.
  • Generative views still contain generated geometry, but the links are removed (the broken links icon is displayed in the tree).
  • If its 2D component reference is not accessible, a 2D component is imported but isolated.

V5 Native Geometry Usage After Import

This section describes the different compatibility statuses of V5 CATParts in the 3DEXPERIENCE environment.

For more information about standard behavior, see Social and Collaborative: Global Design Management: Coexistence and Migration: ENOVIA VPM V5 Coexistence and Migration: Understanding Coexistence and Migration: V5 Native Geometry Usage.

There are two cases of V5 native geometry import as follows:

  • When a V5 native 3D shape representation is imported to 3DEXPERIENCE, the data can be used as if it were native to 3DEXPERIENCE. You can generate drawings for this data using all the 3DEXPERIENCE capabilities in the same session.
  • When a V5 native .CATDrawing file is imported to 3DEXPERIENCE, the data can only be viewed in Explore mode. It cannot be opened or edited.
