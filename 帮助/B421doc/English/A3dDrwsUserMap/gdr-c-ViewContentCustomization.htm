About View Content Configuration

You can configure the graphical properties of generative views to configure the view.

The following topics are discussed:

  • Generation Properties of Views
  • Overloaded Properties
  • Graphical Properties of Generated Geometry and Dress-up

Related Topics
About View Creation

First of all, remember that several elements are taken into account to define the graphical properties of generative views, according to a precise priority order. Some apply at view creation, some apply once a view has been created. This enables you to comply with the chosen standard, while answering your corporate or individual configuration needs. See About the Graphical Properties of Generated Geometry for more information.

The elements listed in this topic are taken into account to configure the graphical properties of generative views once a view has been created.

Generation Properties of Views

You can modify the generation properties of existing generative views, through the Properties dialog box available from the context menu of a given view.

Note: You cannot modify the properties of a view created from another view (i.e. projection view, auxiliary view, section view and detail view). These views have the same properties as their parent view. However, in these views, the properties of 3D points and 3D wireframes can be modified.

Overloaded Properties

You can overload the properties of 3D components in a view generated from a 3D product. Properties are overloaded on a view per view basis, after the view creation.

For more information, see Overloading Element Properties.

Graphical Properties of Generated Geometry and Dress-up

You can change the graphical properties of generated geometry and view dress-up (thread, axis line, center line), in exact views only.

For more information, see Managing the Graphical Properties of Generated Geometry and Dress-Up.
