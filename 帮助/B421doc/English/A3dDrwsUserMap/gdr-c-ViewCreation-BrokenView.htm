About Broken Views

Broken views are used to shorten elongated objects, using two profiles corresponding to the part to be broken from the view extremities.

Miscellaneous Information

This sub-topic provides miscellaneous information about broken views.

  • You can create new breaks in a broken view, but in the same direction, provided the two breaks are not overlapping.
  • You can remove existing breaks by right-clicking the view and selecting View Definition > Unbreak.
  • You can create/modify a broken view which already contains a breakout profile.
  • You cannot create offset section views and clipping views from a broken view. However, you can create detail views in a broken view. For more information about creating details views from broken views, see Creating a Detail View or a Detail View Profile.
  • If a broken view is created after the creation of auxiliary callout, the callout is rendered invalid. The part of the auxiliary callout profile segment inside the broken region zone is seemingly non-existent. This behavior is also true for section callout profiles.

Creation/Modification/Visualization of a Broken Profile in a View Containing a Breakout Profile

You can create/modify/visualize a broken profile in a view which already contains a breakout profile, and, which can be any one of the following types:

  • Breakout in single zone
  • Breakout partially in the broken region zone
  • Breakout across the broken region zone
  • Breakout completely in the broken region zone
  • Breakout in neighboring broken region zone

For more information about breakout profiles, see More About Breakout Views.

Line Type

This sub-topic provides information about line type in broken views.

You can assign a line type to the view to be generated. To do this, go to Me > Preferences > Mechanical > Drafting > View section, click Configure next to View linetype and select the required option from the dialog box.

Propagation of Broken Specifications

This sub-topic provides information about the propagation of broken specifications to other views.

Propagation of Specifications: Definition

Propagating a specification means generating a view (B) from another view (A) on which you previously performed an operation, and including this operation in the generated view (B).

For example, you create a broken or breakout view (view A). When you select Propagate Broken or Propagate Breakout, you can then generate a projection view (view B) with the broken/breakout area.

How to Propagate Broken Specifications

Broken specifications can be propagated during the creation of projection or auxiliary views. This section explains how to propagate broken specifications.

To propagate the specifications, select Propagate Broken in the Creation Options panel, in the Propagate Operator Specifications list.

Important:
  • Broken specifications are not propagated to detail views.
  • You can only perform a propagation from a broken view if the projection direction is perpendicular to the direction of the broken view.
