Overview

A detail view is a partially generated view that shows only what is necessary in the clear description of the object. Note that the Detail View command uses a Boolean operator from the 3D whereas Quick Detail View computes the view directly from the 2D projection. The representation is therefore different.

Note: In detail views and detail view profiles, the dress-up elements generated from the 3D are identified and represented, whereas they are not when generating the quick detail view and quick detail view profiles.

Miscellaneous Remarks About Detail Views and Detail View Profiles

This sub-topic makes miscellaneous remarks about detail views and detail view profiles.

  • If you create a detail view from a section view and subsequently modify the cutting profile in the section view, in such a way that the detail view is no longer accurate, the detail view cannot be updated so as to take into account the changes in the cutting profile.
  • If you create a broken view from a view which contains a detail view callout, the detail view callout will not be broken in the broken view. In other words, the circular shape of the callout will not be modified, even if the callout overlaps the break profile. As for the detail view, it will not be modified by the break operation.

Miscellaneous Remarks About Quick Detail Views and Quick Detail View Profiles

This sub-topic makes miscellaneous remarks about quick detail views and sketched quick detail profiles.

  • If the view which defines the sketched quick detail profiles is modified, the quick detail view is simultaneously updated.
  • If the view which defines the sketched quick detail profiles is removed, the quick detail view recovers its original definition.
  • The view update may be quicker in some cases (depending on the geometry contained in the view which defines the quick detail profile).
  • The whole detail profile is always drawn in the quick detail view.
  • A side representation of threads is available only if threads are first generated in the view which defines the quick detail view.
  • Avoid creating a quick detail view from an aligned section view/cut as it may result in wrongly generated axis lines.

The quick detail view is updated as a standalone view in the following cases:

  • The view which defines the sketched quick detail profiles is locked or not up-to-date.
  • The quick detail view projection mode is different from the projection mode of the view which defines the quick detail profile (i.e. “exact” vs. “approximate”).
  • The view which defines the quick detail view is already a detail/quick detail view or already owns a clipping/quick clipping specification. Therefore, you cannot create a clipping/quick clipping view profile on a detail/quick detail view.

Detail View Profiles Associativity

You can create a detail view profile associative to a 3D geometry, so that even if the geometry is moved, modified or set to the hidden mode, the detail profile will remain associative to the 3D geometry. However, if the associated 3D geometry is deleted, the associativity will be lost.

To create an associative detail profile, you need to:

  1. Sketch the required detail profile.
  2. Select a 2D generative edge in the defining view.
  3. Select a point to denote the location of the resulting detail view.

You need to select the reference 2D generative element to apply the associativity prior to denoting the location at which the resultant detail view is desired to be created. The associativity is created between the corresponding 3D object on the 3D edge/face representing the selected 2D generative element and the starting point of the detail profile.

As soon as the associativity is created, a message appears in the top right corner of the screen informing that an associative detail profile has been created. This message disappears automatically once you indicate the location of the resultant detail view by clicking at the appropriate location in the sheet.

The resultant detail view is associative to the 3D geometry corresponding to the selected 2D generative element. Therefore, if you modify this associated 3D geometry, that modification will be reflected in the resultant detail view when the defining view and the detail view are updated. However, if this 3D object is deleted or not loaded in the session, the associativity will be lost.

Important: Only a 2D generative edge\face in the defining view is considered as a valid reference element to define the associativity. If you select any other element in the defining view, it will not be considered as a valid input and no associativity will be created in such a case.

About Graphic Dress-Up Properties of Detail View Profiles

If you modify the graphic dress-up properties of detail operator profile lines in the detail view, they remain persistent even after updating the view and get applied to all the remaining profile lines of that detail view.

Important: Persistency of graphic dress-up properties is available only for views created using the Exact or the Approximate mode of view projection.

If you select any profile line of a detail operator profile in the detail view, and modify its graphic properties such as color, linetype, linethickness, delete, layer and hide/show, and update the view or use c:force update, then all the remaining profile lines of that detail operator in that detail view are assigned with the new graphic dress-up properties.

Note: The graphic properties of the detail operator profile lines in detail view modified through the Properties or through the Graphic Properties, take precedence over overloading of the graphic properties from XXX object > Overload Properties.

Switching Between Exact and Approximate View Modes

  • The graphic dress-up properties for the operators are maintained independently for Exact and Approximate view modes. Therefore, if you switch between these two view modes, you can visualize the latest set of modified graphic dress-up properties for those particular operator profile lines in that particular view mode. However, if there is no last modification available for those operator profile lines for that view mode, then the default properties set in the standard for that operator in that view mode are seen.
  • For example, you can assign one set of graphic dress-up values for the detail operator in exact view mode and another set of graphic dress-up values for the same detail operator in approximate view mode. When you switch between exact and approximate view modes, you can visualize the profile lines of that same detail operator differently.

Editing the Detail Operator Profile

Graphic dress-up properties assigned to profile lines of a detail operator are retained even after performing edition of the same operator.
