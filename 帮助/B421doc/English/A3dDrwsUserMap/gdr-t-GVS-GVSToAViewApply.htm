Managing Generative View Styles

Generative view styles are defined by an administrator and specify the appearance and behavior of a generated view. You can, manage these generative view styles by applying it to another view or by overriding its parameters.

This task shows you how to:

  • Apply Generative View Styles to a View
  • Override Generative View Style Parameters

Applying the Generative Style of a View to Another View

Apply Generative View Styles to a View

Generative view styles are defined by an administrator and specify the appearance and behavior of a generated view. You can, however, apply a generative view style to a view which was created without one.

Before you begin:
  • Open a drawing representation.
  • Create a view without a generative view style.

  1. Go to Me > Preferences > All Preferences > 3D Modeling > Mechanical Systems > Drafting > Administration tab, and clear the Disable generative view style usage check box.

    This activates the generative view style functionality.

    Important: If you clear Disable generative view style usage check box in Me > Preferences > All Preferences > 3D Modeling > Mechanical Systems > Drafting > Administration tab, under Generative view style, then the 3D colors check box is not available in the Properties > Generation tab any more.

  2. Right-click the front view and select Front view object > Generative View Style > Set View Style. The Choose a style dialog box appears, displaying the list of available generative views styles.

    Important: The generative view styles available in the list depend on pre-defined set of parameters by the administrator.

  3. Select a style and click OK.

  4. To update the view, click Update .

    The view is updated. However, the selected style is not visible in the view yet. For the moment, the geometry is displayed in a specific color, because the original parameters of the view overload the generative view style parameters. (This means that the parameters of the selected generative view style have taken on the value of the parameters that were defined in the view before you applied the style to it.)



    You now need to reset the parameters of the generative view style to their original values so that the new style to be visible in the view.

  5. Right-click the front view again and select Properties.

    is displayed.

  6. In the Properties dialog box, select the Generation tab, under Dress-up, an asterisk * appears in front of the parameters which are considered as being overloaded.

    The list in the Generative view style shows the generative view style which is applied to the view.

  7. Click Reset to style values , and then click OK to validate.

    The view is updated once again and the parameters defined in the selected generative view style are reset to their original values and are applied to the view.



    In our example, hidden edges are projected in orange, with a different linetype and thickness than previously. Note that you may obtain a different result. This result depends on the parameters defined in the generative view style you selected.

Override Generative View Style Parameters

You can override the generative view style parameters defined locally on the view to the parameters defined in the generative view style referenced by the view.

  1. Go to Me > Preferences > All Preferences > 3D Modeling > Mechanical Systems > Drafting > Administration tab, and select the Enable parameter override check box and click OK.

    This lets you override the generative view style parameters.

  2. Create a front view using a generative view style.



  3. Right-click the view and select Properties > GVS tab.

    It shows the name of the current generative view style with status. You can apply another generative view style by selecting it in the list.

  4. Click Override GVS Parameters.

    The GVS Editor (XXX.xml) dialog box appears (where XXX denotes the name of the generative view style being edited). You can select a required parameter using search, filter, or by selecting it in the tree structure.

  5. Optional: Select a parameter using Search:

    1. Type a required parameter name in the Search box.

      The Search box displays the matching results as you type in the search string.

    2. Select the parameter from the search results.

      The left pane displays that parameter in the tree structure. Simultaneously, the parameter values are displayed in the right pane in the Parameter tab. You can change the parameter value as per your requirement.

  6. Optional: Select a parameter using Filter:

    1. Select a required option in the Filter list. For example, select Drawings.

      By default, the Filter list shows name of the current generative view style as the first option.

      The left pane shows Drawings node and its sub-nodes.

    2. Select the required parameter in the tree structure.

      The Parameter tab in the right pane displays all the relevant parameters. You can change their values as per your requirement.

  7. Navigate through the tree structure and click the required node. For example, click Drawings node and select VividEdges under GeneratedGeometry.

    All the parameters related to vivid edges are displayed in the Parameter tab available in the right pane.

  8. Change the vivid edges Color, Linetype or Thickness value for example.

    As soon as you change the values, an asterisk (*) appears in front of the changed parameter. Additionally, the Reset appears in front of each changed parameter.

  9. Optional: Click Reset to reset the parameter value to the originally defined one.

    The asterisk (*) and the Reset disappear.

  10. Select the All Overridden Parameter tab.

    It displays the list of overridden parameters. You can change the values again or reset the values by clicking Reset.

  11. Optional: Click Reset All to reset all the overridden parameters.

    The parameters are reset, and, if required, you can define the new values by following the steps from 6 through 8.

  12. Click OK in the GVS Editor (XXX.xml) dialog box.

  13. Click OK in the Properties dialog box.

    The generative view style parameters are overridden locally.
