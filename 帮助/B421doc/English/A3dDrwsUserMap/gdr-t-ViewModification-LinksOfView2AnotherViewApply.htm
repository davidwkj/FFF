Modifying the Link of View to Another View

You can modify the link of a view to another view. This is useful when you create the instance of a product in an assembly.

This task shows you how to:

  • Link a View to Another View of the Same Root Product
  • Link a View to Another View of a Different Root Product

Before you begin: Open a 3D representation. Make sure no drawing is already open. Create new drawing representation, using the All Views wizard.
Related Topics
Modifying Links of a View

Link a View to Another View of the Same Root Product

You can apply a link of a view to another view of the same root product.

  1. Right-click the front view.

  2. Select Front View object > Modify Links.

  3. In the 3D area, press Ctrl, then select several 3D elements.

    Back in the drawing, the 3D elements are listed in the Link Modification dialog box.

  4. Click Add all and then OK.

    The Link Modification dialog box is closed and an Update icon appears in the tree under the front view node.

  5. Click Update.

    The front view is updated accordingly.

  6. Right-click the front view and select FrontView object > Apply Links To.

  7. Select the bottom view.

    An Update icon appears in the tree under the bottom view node.

  8. Click Update .

    The view is updated accordingly.

Important:
  • Only one view can be selected.
  • Do not try to apply the same links to the view.

Link a View to Another View of a Different Root Product

You can change the associated root context of a view by applying the links of another view.

Before you begin: Create views from different products (product A and product B for example) in the same drawing representation.

  1. Right-click the view and select FRONT object > Apply Links To.

  2. Select the view from another product (product B for example).

    • A message appears informing that the target view is not linked to the same product. All the internal links of this view have been isolated (dimensions, overloaded properties, ...).
    • When you switch from a product instance to the reference of this product instance, the links are retained. The dimensions, overload properties, annotations, and other properties are still associative. A message appears informing that the views linked to the same subassembly are modified.

      Any views which belong to the same instance, product A, but are not selected for the link update are also linked to the reference of the instance, product B, in this process.

  3. Click Update.

    The view is updated and it is now linked to the context of another root product.
