Computing the Drawing PLM Update Status

You can compute the drawing PLM update status that consists in querying the PLM database to know whether a drawing is up-to-date or not, as well as the time and date at which the status was computed.

Important: The PLM update status can only be considered valid at the time of computation.

This task shows you how to:

  • Compute the Drawing PLM Update Status Using the Drawing Update PLM Batch
  • Compute the Drawing PLM Update Status Using the Search Results Reader

Compute the Drawing PLM Update Status Using the Drawing Update PLM Batch

You can compute the drawing PLM update status using the drawing update PLM batch.

Before you begin:

From the Tools section of the action bar, click Utility and log on. The Batch Monitor appears.

  1. From the list of available batches in the batch monitor, double-click Drawing Batch.

    The Drawing Batch dialog box appears.

  2. In the Selected Operation area, select the impacted views and the action to launch from the appropriate drop-down list.

    For this scenario, select Views from whole drawing and Update status.

    Important: These parameters will be applied to every drawing selected in steps 3 and 4. However, you can configure these parameters afterwards for one or several drawings. For this, select (or multi-select) a drawing in the list, then change its parameters in the Selected Operation area.

  3. Click to select the drawing representations you want to update or isolate by batch.

    The [Name_of_the_provider_you_are_connected_to] Object Management dialog box appears. For more information about this dialog box, see Searching for Objects.

  4. Define your search criteria, click Apply, then OK .

    The selected files are now listed in the Object Selection table, along with the impacted views and action to execute.

  5. Optional: Repeat steps 3 and 4 to include files from another location.

    Note: You can remove files from the list of the files to update. To do so, select the unwanted file and click .

  6. Optional: In the Report directory, modify the path of the directory in which the reports will be stored.

  7. Optional: Modify the name that will be used for the report in the Report name box. By default, the global report dealing with all the documents that have been processed is named GlobalResults.htm.

  8. For this scenario, select the Run local.

    Your drawing PLM update status parameters are now defined.

  9. Optional: Click Save if you want to save these parameters to re-use them later on.

    1. Browse to the directory in which you want to save the batch parameters file.
    2. Specify a file name in the appropriate box.
    3. Click Save.

    Your parameters are saved in an xml file.

  10. Click Run to run the batch immediately.

    The batch execution starts.

    Note: You could also click Cancel to close the Update Batch dialog box and run the batch later on.

  11. Click the Processes tab during or after the batch execution for more information on the process such as the batch name, the batch identification number, name on the host computer (if you run the batch on a remote computer), the status (in progress or ended) as well as the start and end times.

  12. To display the batch report and get a PLM update status for each updated drawing or view, double-click the batch in this list.

    The batch report is displayed in your browser. You can then display the PLM update status for each listed object.

  13. When the batch execution is over, you can close the Batch Monitor.

Compute the Drawing PLM Update Status Using the Search Results Reader

You can compute the drawing PLM update status using the search results window.

Before you begin: From the top bar, in the Search Criteria box, enter a search string and click Search .

The list of all objects corresponding to your search criteria appears.

  1. In the Search Results panel, right-click the drawing representation whose PLM update status you want to compute.

  2. Click Update status.

    The PLM update status computation starts.

When the computation is over, the PLM update status of the selected representation is displayed in an information message.
