Creating a Table

General Information

This sub-topic provides general information about tables.

  • The height of a table cell depends upon the text properties and the vertical margin. The margin related attributes can only be positive values. See Text Properties of a Table Cell.
  • The width of a table cell corresponds to the height of a string multiplied by 5.
  • The smallest table which can be created is of one cell, which has one row and one column.
  • The biggest table which can be created is of 10000 cells (example: 102 columns and 98 rows). However, capacity of the computer (available memory and CPU) may induce a lower limit.

Text Properties of a Table Cell

The height of a table cell depends on the text properties, such as font type, font size, etc. In other words, these properties define the height of a table cell. The default values are set in Me > Preferences > Standards > Drafting > [StandardName] > Styles > Table. However, you can change these values if required.

Table Creation Modes

There are three table creation modes which help you design your table as per your requirement based on your selected reference.

The table creation modes are displayed when you place the pointer over the balloon after you selected a reference:



  • Creates a table aligned on reference : If the reference is a line, the created table is positioned on the point of indication or point of reference on the line. If the reference is a table, the created table is positioned on the nearest border point of the table.

    The table is created with variable number of rows and columns as in case of table creation through point indication.



  • Creates a table aligned on the nearest extremity : If the reference is a line, the created table is positioned on the nearest extremity of the line. If the reference is a table, the created table is positioned on the nearest corner point of the table.

    The table is created with variable number of rows and columns as in case of table creation through point indication.



  • Creates a table constrained on reference : If the reference is a line, the created table size is constrained with respect to the size of the reference line and aligned on it. This mode is unavailable for multiple bills of materials generated with the Multiple BOM command. This mode is available for a single table generated using Multiple BOM.

    For example, if the reference is a horizontal line, the created table will have a table width (or number of columns) equal to the length of the selected line. In case of a vertical reference line, the created table will have a table height (or number of rows) equal to the length of the selected line.



    If the reference is a table, then the created table is constrained according to the reference table's height or width. However, note that, unconstrained dimension of the table (row or column) remains variable and can be changed with the pointer movement. In this case, the table orientation handles are not displayed and the table is created directly.

Important:
  • If the size of the reference is smaller than the size of one cell, then, only two table creation modes are displayed, aligned on reference and aligned on the nearest extremity. When the reference size is larger than the size of 100 cells, then the table is created by default on the selected point of reference which is similar to align table on reference mode. However, in this case, the table creation modes are not displayed.
  • If the number of columns (or rows) obtained after selecting the reference object is not as desired, it is still possible to increase or decrease them using the dialog box.
  • If the reference object and the created table belong to the same view, then, an associative positional link is set between the created table and the reference object. However, if they belong to the different views, no associative positional link is set.
