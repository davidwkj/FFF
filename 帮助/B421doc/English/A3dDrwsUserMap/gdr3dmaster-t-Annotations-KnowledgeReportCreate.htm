Inserting a Knowledge Report in a Representation

You can insert a knowledge report in a as a table. However, this table is not associative to the changes performed in either the template or the attributes reported.

Before you begin:
  • Open a representation and an assembly product.
  • Create a knowledge report template with preferred attributes.

    For more information about creating a report template, see Quality Rules Reuse User's Guide: Working with Report Templates: Creating a Report Template.

  1. From the Annotation section of the action bar, click Knowledge Report Table .

  2. In the Report Generation dialog box, click Select.

    1. In the XML Document Chooser dialog box, select the From Search tab, enter the report name and click Search.

      You can select the created knowledge report template from this dialog box.

      Note: You can also search the report by selecting the From File tab and clicking Import.

      Matching results are displayed in the list below.

    2. Select the required report template and click OK.

      The selected report appears in the Report Name box.

  3. Choose Selected objects.

    Note: You can also select All objects in session to get the report of all the objects of your representation.

  5. Click Generate Report.

  6. Click at the required location in the drawing to position the knowledge report.

    The report is created in the active view. (It is created in the sheet itself, also known as the main view if no view is preselected).

    The table is inserted based on the associated favorite table style. However, you can define a specific table style for the knowledge report.
