Working with Standards and Generative View Styles

You can customize standards and generative view styles and set their locations. The procedure is the same for standards or Generative View Styles (GVS). When applicable, differences are notified.

This task shows you how to:

  • Set the Location of Standard/GVS Files in a Custom Directory
  • Set the Location of Standard/GVS Files Using Variables
  • Configure and Define Standards or GVS

Related Topics
About the Location of Standard or Generative View Styles Files

Set the Location of Standard/GVS Files in a Custom Directory

You can set the location of standard or generative view style files in a custom directory.

If you want to place all configured drafting standards or GVS in a custom directory (mydirectory for example), proceed as follows:

  1. Create a directory named as you like (mydirectory, for example).

  2. Create a sub-directory under this directory, called either drafting or generativeparameters, depending on the context.

  3. Place the XML files containing your configured drafting standards or GVS in mydirectory\drafting or mydirectory\generativeparameters.

Set the Location of Standard/GVS Files Using Variables

You can set the location of standard or generative view style files using variables.

If you have not yet configured your XML standard files, you can set the location of your files using a variable. In this case, proceed as follows:

  1. Create a directory named as you like (mydirectory, for example).

  2. Create a sub-directory under this directory, called either drafting or generativeparameters, depending on the context.

  3. Set the CATCollectionStandard variable to mydirectory. After you have configured the XML standard files, the standard editor will then save them in mydirectory\drafting or mydirectory\generativeparameters.

    Notes:

    • If the CATDefaultCollectionStandard and the CATCollectionStandard variables both contain an identically-named standard, it is always the standard found in CATCollectionStandard which will be used.

    • If two directories referenced by the CATCollectionStandard and CATDefaultCollectionStandard variables contain identically-named standard files, it is always the standard in the directory listed first which will be used.

Configure and Define Standards or GVS

You can configure and define standards or generative view styles.

  1. To edit and save standard files or generative view style files in Me > Preferences > Standards, set up the CATReferenceSettingPath variable.

    For more information, see Installation and Configuration User's Guide: Configuring the Software: Configuring Environments.

    Note: You can customize standards provided that the business role used when logging on allows you to do so.

    A role is one of the components of the security context used to log on. For content security purpose, each user is assigned to one or several secured collaborative spaces with a specific role, to manage the different categories of end user content.

    If you are logged as an owner (internal name is VPLMProjectAdministrator), you are allowed to customize the standards. In that case, the Save As New is available to let you save your modifications in a new .xml file.

  2. Set up the CATCollectionStandard environment variable as explained above. If none of the conditions are respected, a warning message will appear to let you know that you will neither be able to modify nor save the XML files. Modify the Drafting standards or the generative view styles as appropriate.

  3. Click Save As or OK to store your modifications.

  4. Click Cancel to exit.

    Once the standard files or the generative view style files have been configured and saved, they can be used in a session in normal mode.
